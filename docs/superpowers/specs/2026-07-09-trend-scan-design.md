# 趋势质量扫描（Trend Scan）设计文档

*讨论时间：2026-07-09*
*相关背景：滑动窗口验证策略讨论记录.md（诊断出参数网格优化容易过拟合的问题）*

## 背景与目标

用户不再追求全自动交易，而是想要一个**决策辅助看板**：把当前趋势最"稳"、适合按用户自选周期（日内/周线，不做区分，用户自己选周期）操作的品种按评分列出来，并给出止盈止损建议和操作建议，最终由用户自己判断是否手动下单。

前序讨论已经诊断出：现有「多品种扫描」（`ScanTab.vue`）的评分依赖对 119 组 MA 参数网格做历史收益最优化（`App.vue` 的 `handleScan`），这种"用历史收益率挑参数"的方式在短窗口上极易过拟合，选出的参数往往是拟合了噪声而非真实趋势规律。本功能需要一套不依赖回测参数优化、只看"当下趋势质量"的独立评分方式。

## 范围

- 新增一套评分引擎，评估"当前趋势是否清晰、稳定，适合操作"，输出 0-100 分 + 方向（多/空/中性）。
- 新增止盈止损建议，基于近期价格结构（摆动高低点），而非固定百分比。
- 复用现有「多品种扫描」的选币、周期、K线数量、排序、筛选 UI 框架。
- 不做自动化下单，不做新的历史回测统计（历史收益率/胜率/回撤这些字段在新引擎里不存在，直接从展示中去掉，不伪造）。

## 架构

新增 `freqtrade-webui/src/composables/useTrendScore.ts`，只做纯计算：

```
App.vue（新增 handleTrendScan）
  → useDataFetch.loadData(pair, timeframe, limit)        [复用现有，不改]
  → useTrendScore.scoreSymbol(dates, data)                [新增，纯函数]
      ├─ calculateTrendQuality()  → { score, direction, level, adx, efficiencyRatio }
      └─ calculateSwingSLTP()     → { stopLossTight, stopLossWide, takeProfit, riskRewardTight, riskRewardWide, isSwingBased }
  → 汇总成 TrendScanResult[]，交给 ScanTab（改造后）展示
```

**关键边界**：`useTrendScore.ts` 从 `useBacktest.ts` **只借用** `calculateMA` / `calculateADX` 两个纯指标函数（这两个函数本身没有过拟合问题，问题出在"用回测收益率去挑参数"那一层）。`useTrendScore.ts` **不引用** `runBacktestWithParams` 及任何参数网格优化逻辑，从架构上隔绝过拟合风险。副作用是计算量从每品种 O(119 次回测) 降到 O(1 次指标计算)，扫描速度会明显变快。

## 趋势质量评分算法

不依赖历史回测收益，只用当下的、参数固定（不调参）的经典指标组合出 0-100 分：

| 维度 | 计算方式 | 作用 |
|---|---|---|
| 趋势方向一致性 | 固定用 MA5/MA10/MA20/MA50（不做参数搜索），判断是否完全多头排列（5>10>20>50）或完全空头排列 | 排列越整齐，趋势越"干净" |
| 趋势强度 | 复用 `calculateADX`（14周期标准参数，不调） | ADX 越高，趋势力度越强，越不容易是震荡假信号 |
| 趋势效率比（Efficiency Ratio） | `\|close[i] - close[i-N]\| / Σ\|close[j]-close[j-1]\|`（Kaufman 效率比，N≈20），比值越接近 1 说明走势越顺畅干净，越接近 0 说明来回震荡 | 直接量化"趋势是否稳"，是成熟、无需调参的经典指标 |
| 波动率状态 | ATR(14) 相对自身历史分位数，异常偏高则扣分 | 避免选中正在剧烈甩针、隔日跳空的品种 |

四项加权求和得 0-100 分；方向由均线排列决定；均线排列与效率比方向不一致时（如均线多头排列但效率比很低），视为"趋势不干净"扣分，而非矛盾判定报错。分档沿用现有 `level`（strong/moderate/weak/neutral）展示风格，文案改为"趋势稳定，适合操作 / 趋势尚可，谨慎参与 / 趋势不明，观望"，不再出现"历史胜率"相关表述。

## 止盈止损（近期摆位高低点，含抗插针设计）

**摆动点识别（强度过滤）**：不用任意 2 根K线的简单分型，改为在更长回看窗口（约 100-150 根）内寻找**局部极值 + 历史上被价格多次尊重（触碰后反弹/受阻 ≥1 次）**的位置，过滤掉随手就有的噪声摆动点，只保留有真实防守意义的结构位。

**海龟汤（Turtle Soup）风险的应对**：贴着摆动点放止损容易被针对性插针扫损后反向。应对措施：

1. 上述摆动点强度过滤本身就减少了"伪结构"被当成止损参考点的情况。
2. **缓冲距离 = 历史插针深度统计**：扫描时统计该品种近期"价格刺穿摆动点又收回"的插针幅度，止损缓冲设到能扛过该品种典型插针深度之外；数据不足（新币种/样本少）时退化为 ATR 倍数（2-3×ATR）兜底。
3. **双档止损展示**：每个信号同时给出
   - 贴近档：摆动点 + 最小缓冲，盈亏比更好看，但容易被插针扫到；
   - 抗插针档：摆动点 + 插针深度缓冲，止损更宽，盈亏比稍差，但更能扛住插针。
   两档都计算并展示各自的盈亏比，由用户按品种流动性和自身风险偏好选择。
4. **收盘确认价（仅展示提示）**：额外算一个"确认破位价"（收盘价越过止损线才算真破位），仅作为界面提示，明确标注这不能当挂单价格用——真实止损单按价格触及（含影线）成交，和"收盘确认"是两回事，UI 文案需要写清楚以避免误解。

**做多信号**：止损 = 下方最近摆动低点（按上述两档处理）；止盈 = 上方最近摆动高点。
**做空信号**：方向相反。
**兜底情况**：近期窗口内找不到明显摆动点时（如刚突破新高/新低），退化为 ATR 固定倍数估算止盈止损，结果中标记 `isSwingBased=false`，UI 显示 ⚠"按波动率估算"。
**盈亏比过滤**：盈亏比低于 1.2 的信号，即使趋势分数高，也在展示上标记"低吸引力"提示，不做剔除，由用户自行判断。

## UI 改动（复用 ScanTab 框架）

- `types/index.ts` 新增 `TrendScanResult` 类型（与现有 `ScanResult` 并存，不改动原类型）：
  ```
  pair, timeframe, trendScore, direction, level,
  adx, efficiencyRatio, volatilityState,
  stopLossTight, stopLossWide, takeProfit,
  riskRewardTight, riskRewardWide, isSwingBased
  ```
- `ScanTab.vue`：
  - 新增列：止损（贴近/抗插针两档）、止盈、盈亏比；`isSwingBased=false` 的行止损单元格加 ⚠ 提示"按波动率估算"
  - 排序默认键从 `totalReturn` 换成 `trendScore`
  - 保留现有筛选按钮（强烈建议/建议/轻仓/观望），文案去掉"历史胜率"相关表述
  - 移除"历史收益/胜率/回撤"三列（新引擎不产出这些数据，不伪造、不留空）
- `App.vue`：新增 `handleTrendScan`，结构参考现有 `handleScan`，但去掉参数网格搜索的双层循环，直接对每个选中品种调用 `useTrendScore.scoreSymbol()`。
- `ParamsPanel.vue`：新增按钮"📈 趋势扫描"（区别于现有"📊 多品种扫描"），复用已有选币/周期/K线数量控件；MA/止损止盈/ADX 阈值等输入框对趋势扫描不生效，需在按钮旁加提示文案："趋势扫描使用固定标准参数，不受下方回测参数影响"。

## 错误处理与边界情况

- **数据量不足**：效率比/MA50/摆动点识别至少需要 100-150 根K线，不足时该品种标记"数据不足，无法评分"，不强行输出不可靠分数。
- **OKX API 失败回退 mock 数据**：复用现有 `useDataFetch` 的 mock 兜底；mock 数据是随机游走，算出的摆动点/效率比没有实际意义，沿用现有 `isRealData` 标记，在结果表中对非真实数据整体加 ⚠"模拟数据，仅供界面预览"提示。
- **找不到任何摆动点**：走 ATR 兜底，`isSwingBased=false`（已在止盈止损部分覆盖）。
- **单品种计算异常不影响整体**：参考现有 `handleScan` 的 try/catch 单品种隔离模式（`App.vue:297-299`），单个品种失败不影响其他品种继续出结果。

## 测试策略

项目当前未配置测试框架（`package.json` 无 vitest/jest），是否引入测试框架是本次改动范围外的独立决定。本次验证方式：

- 手动 `yarn dev`，选几个明显在趋势中的品种和几个明显震荡的品种，对比新引擎打分是否符合直觉。
- 抽查止盈止损：把建议价格标回K线图，肉眼确认摆动点位置和双档缓冲是否合理。
- 纯展示、无自动交易，无资金风险，不需要额外回归测试保护。
