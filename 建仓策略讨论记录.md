# 多品种扫描建仓策略讨论记录

## 背景

用户希望基于多品种扫描结果制定合理的建仓策略，核心问题是：**扫描结果中的哪些参数可以作为可靠的购买信号？**

---

## 一、扫描结果字段分析

当前多品种扫描结果包含以下字段：

| 字段 | 说明 | 信号价值 |
|------|------|---------|
| 排名 | 按收益率排序 | 辅助参考 |
| 交易对 | 币种 | 基础信息 |
| 周期 | 时间框架 (1H/4H/1D) | 影响策略频率 |
| 推荐操作 | 做多/做空/观望 | **核心信号** |
| 快线 (MA) | 短期均线周期 | 参数优化结果 |
| 慢线 (MA) | 长期均线周期 | 参数优化结果 |
| 预期收益 | 历史回测收益率 | 盈利能力 |
| 胜率 | 盈利交易占比 | **稳定性指标** |
| 最大回撤 | 最大资金亏损 | **风险控制** |
| 趋势强度 (ADX) | 趋势强度值 | 过滤噪音 |
| 信号天数 | 距离交叉的K线数 | **新鲜度指标** |

## 二、推荐操作计算逻辑

### 原始逻辑（有问题）

```javascript
// 只在交叉发生的当根K线成立，之后显示"观望"
if (prevFast <= prevSlow && currFast > currSlow && currentAdx > adxThreshold) {
    trendSignal = 'long';  // 金叉
} else if (prevFast >= prevSlow && currFast < currSlow && currentAdx > adxThreshold) {
    trendSignal = 'short'; // 死叉
}
```

**问题**：严格交叉条件只在交叉瞬间成立，后续K线都显示"观望"。

### 修复后逻辑

```javascript
// 根据当前快慢线关系判断状态
if (currFast > currSlow) {
    trendSignal = 'long';
    // 往前找交叉点计算信号天数
    for (let i = lastIdx; i > 0; i--) {
        if (pf <= ps && f > s) {  // 找到金叉点
            signalAge = lastIdx - i;
            break;
        }
    }
} else if (currFast < currSlow) {
    trendSignal = 'short';
    // 同理计算死叉天数
}
```

**改进**：
- 只要快线 > 慢线就认为处于金叉状态
- 通过往前遍历找到实际交叉发生的K线位置
- 计算 `signalAge = lastIdx - i`（距离现在多少根K线）

## 三、建仓信号分级系统

结合**信号天数**和**ADX趋势强度**，将推荐操作分为三级：

### 1. 最佳建仓时机（信号 ≤2K 且 ADX>20）

```
显示：🟢 做多/做空 (最佳)
颜色：绿色 + 粗体
建议：✅ 立即重仓建仓
理由：
- 信号刚产生，趋势刚开始
- ADX>20 确认趋势强度足够
- 止损位清晰，风险可控
```

### 2. 可建仓时机（信号 3-5K）

```
显示：🟢 做多/做空 (可建仓)
颜色：绿色
建议：⚠️ 轻仓试探或等待回调
理由：
- 趋势已确认，但成本略高
- 可能错过最佳入场点
- 需要更严格的止损
```

### 3. 观望时机（信号 >5K 或 ADX≤20）

```
显示：🟡 观望 (金叉/死叉已久)
颜色：金色/灰色
建议：❌ 避免追高，等待下一个信号
理由：
- 信号已过期，可能追高
- 风险收益比变差
- 回调风险增大
```

## 四、ADX阈值配置

### 参数调整

| 参数 | 原值 | 新值 | 说明 |
|------|------|------|------|
| ADX阈值 | 20 | 5 | 更容易触发交易 |
| 最小值 | 10 | 0 | 可完全关闭ADX过滤 |
| 提示 | 无 | "设为0关闭ADX过滤" | 用户引导 |

### ADX作用

- **ADX > 30**：强趋势，信号可靠
- **ADX 20-30**：中等趋势，可用
- **ADX < 20**：弱趋势/震荡，建议观望
- **ADX = 0**：关闭过滤，纯看金叉死叉

## 五、建仓策略方案

### 方案一：保守型（低风险）

```
必要条件：
1. 推荐操作 = "做多 (最佳)" 或 "做空 (最佳)"
2. 胜率 ≥ 50%
3. 最大回撤 ≤ 15%
4. 信号天数 ≤ 2K
5. ADX > 20

可选加分：
- 预期收益 > 20%
- 交易次数 ≥ 10（样本充足）
```

### 方案二：平衡型（收益风险比）

```
必要条件：
1. 推荐操作非"观望"
2. 预期收益 / 最大回撤 ≥ 2
3. 胜率 ≥ 45%
4. 信号天数 ≤ 5K
5. ADX > 15

可选加分：
- 胜率 > 60%
- 最大回撤 < 20%
```

### 方案三：激进型（高收益）

```
必要条件：
1. 推荐操作明确（非观望）
2. 预期收益 ≥ 30%
3. 胜率 ≥ 40%
4. 交易次数 ≥ 5

可选加分：
- 最大回撤 < 30%
- ADX > 25（强趋势）
```

## 六、综合评分算法（满分100分）

### 评分维度

| 指标 | 权重 | 评分标准 |
|------|------|---------|
| **信号新鲜度** | 30分 | ≤2K:30分, ≤5K:20分, ≤10K:10分, >10K:5分 |
| **胜率** | 25分 | ≥60%:25分, ≥50%:20分, ≥40%:15分, ≥30%:10分, <30%:5分 |
| **最大回撤** | 20分 | ≤10%:20分, ≤15%:15分, ≤20%:10分, ≤30%:5分, >30%:0分 |
| **ADX趋势强度** | 15分 | >30:15分, >20:10分, >15:5分, ≤15:0分 |
| **预期收益** | 10分 | ≥50%:10分, ≥30%:8分, ≥10%:5分, ≥0:3分, <0:0分 |

### 操作等级划分

| 分数 | 等级 | 做多显示 | 做空显示 | 建议 |
|------|------|---------|---------|------|
| ≥80 | excellent | 🟢 做多 (最佳) | 🔴 做空 (最佳) | ✅ 立即重仓 |
| 60-79 | good | 🟢 做多 (推荐) | 🔴 做空 (推荐) | ✅ 正常建仓 |
| 40-59 | hold | 📈 多仓持有 | 📉 空仓持有 | ⚠️ 可持有/轻仓 |
| <40 | neutral | ⚪ 观望 | ⚪ 观望 | ❌ 不建议操作 |

### 算法实现

```javascript
function calculateSignalScore(params) {
    const { trendSignal, signalAge, adx, winRate, maxDrawdown, totalReturn } = params;
    
    if (trendSignal === 'neutral') return { score: 0, action: '观望', level: 'neutral' };
    
    let score = 0;
    
    // 1. 信号新鲜度 (30分)
    if (signalAge <= 2) score += 30;
    else if (signalAge <= 5) score += 20;
    else if (signalAge <= 10) score += 10;
    else score += 5;
    
    // 2. 胜率 (25分)
    if (winRate >= 0.6) score += 25;
    else if (winRate >= 0.5) score += 20;
    else if (winRate >= 0.4) score += 15;
    else if (winRate >= 0.3) score += 10;
    else score += 5;
    
    // 3. 最大回撤 (20分) - 回撤越小越好
    if (maxDrawdown <= 0.1) score += 20;
    else if (maxDrawdown <= 0.15) score += 15;
    else if (maxDrawdown <= 0.2) score += 10;
    else if (maxDrawdown <= 0.3) score += 5;
    else score += 0;
    
    // 4. ADX趋势强度 (15分)
    if (adx > 30) score += 15;
    else if (adx > 20) score += 10;
    else if (adx > 15) score += 5;
    else score += 0;
    
    // 5. 预期收益 (10分)
    if (totalReturn >= 0.5) score += 10;
    else if (totalReturn >= 0.3) score += 8;
    else if (totalReturn >= 0.1) score += 5;
    else if (totalReturn >= 0) score += 3;
    else score += 0;
    
    // 确定操作等级
    let action, level;
    if (score >= 80) {
        action = trendSignal === 'long' ? '做多 (最佳)' : '做空 (最佳)';
        level = 'excellent';
    } else if (score >= 60) {
        action = trendSignal === 'long' ? '做多 (推荐)' : '做空 (推荐)';
        level = 'good';
    } else if (score >= 40) {
        action = trendSignal === 'long' ? '多仓持有' : '空仓持有';
        level = 'hold';
    } else {
        action = '观望';
        level = 'neutral';
    }
    
    return { score, action, level };
}
```

### 评分显示规则

| 分数范围 | 颜色 | 样式 |
|---------|------|------|
| ≥80 | 绿色 | 加粗 |
| 60-79 | 绿色 | 正常 |
| 40-59 | 金色 | 正常 |
| <40 | 灰色 | 正常 |

## 八、实际操作建议

### 每日扫描后筛选流程

```
第1步：只看"推荐操作"为 最佳/可建仓 的
       ↓
第2步：按胜率降序排列，筛选前50%
       ↓
第3步：剔除最大回撤 > 20% 的
       ↓
第4步：按预期收益排序，选前5-10名
       ↓
第5步：人工确认K线形态和基本面
```

### 界面使用技巧

1. **点击表头排序**：
   - 先按"信号天数"升序 → 找最新信号
   - 再按"胜率"降序 → 找稳定策略

2. **组合筛选**：
   - 信号天数 ≤2 + 胜率 >50% + 回撤 <15% → 优质标的
   - 收益 >30% + 胜率 >45% + 信号 ≤5 → 高性价比

3. **周期选择**：
   - 1H：短线，信号多但噪音大
   - 4H：中线，平衡型
   - 1D：长线，信号少但稳定

## 九、注意事项

1. **历史不代表未来**：回测收益高不代表未来一定盈利
2. **样本量问题**：交易次数 < 10 的结果可信度低
3. **过拟合风险**：MA参数过于优化可能不适应未来市场
4. **建议组合**：不要单押一个，选3-5个分散风险
5. **信号新鲜度**：金叉/死叉超过5K后，回调风险增大

## 十、技术实现要点

### 信号天数计算

```javascript
// 计算距离最近交叉的K线数
function calculateSignalAge(maFastValues, maSlowValues, currentIdx) {
    let signalAge = 0;
    for (let i = currentIdx; i > 0; i--) {
        const f = parseFloat(maFastValues[i]);
        const s = parseFloat(maSlowValues[i]);
        const pf = parseFloat(maFastValues[i-1]);
        const ps = parseFloat(maSlowValues[i-1]);
        
        // 找到交叉点
        if (pf <= ps && f > s) {  // 金叉
            signalAge = currentIdx - i;
            break;
        }
    }
    return signalAge;
}
```

## 十一、总结

### 核心原则

1. **信号新鲜度优先**：刚产生的金叉/死叉最可靠
2. **趋势强度确认**：ADX>20 确保不是假突破
3. **风险控制**：胜率>50% + 回撤<20% 是底线
4. **分散投资**：不要单押一个交易对

### 最佳实践

- **每天扫描一次**，找信号天数 ≤2 的标的
- **结合多周期确认**：1H金叉 + 4H也金叉 = 强信号
- **设置止损**：单笔亏损不超过本金的10%
- **记录交易**：用于后续策略优化
